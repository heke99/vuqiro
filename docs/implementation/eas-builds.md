# EAS Builds & Internal Testing

Vuqiro uses the managed Expo workflow: native projects are generated by
prebuild/EAS and are **not** committed (`apps/mobile/android|ios` are
gitignored). Config source of truth: `app.json` + `eas.json`.

## Validated in this repo (no credentials required)

- `npx expo-doctor` → **20/20 checks pass** (deps aligned with SDK 57,
  schema valid, no duplicate native modules).
- `npx expo prebuild --platform android` and `--platform ios` generate clean
  native projects with the correct identifiers
  (`com.diversasolutions.vuqiro`) and all permission strings.
- `eas.json` matches the product spec: `cli >= 20.5.0`,
  `appVersionSource: remote`, development (dev-client, internal), preview
  (internal), production (auto-increment) profiles.

## One-time setup (owner) 🔑

```bash
npm i -g eas-cli
eas login                        # Expo account for Diversa Solutions LLC
cd apps/mobile
eas init                        # links the project (writes extra.eas.projectId)
eas credentials                 # iOS: App Store Connect API key; Android: keystore
```

Accounts needed: Expo, Apple Developer (Diversa Solutions LLC),
Google Play Console.

## Build commands

```bash
# Development clients (install on device, connect to `pnpm dev:mobile`)
eas build --profile development --platform ios
eas build --profile development --platform android

# Internal/preview builds (shareable, no dev server)
eas build --profile preview --platform ios
eas build --profile preview --platform android

# Store builds
eas build --profile production --platform ios
eas build --profile production --platform android

# Submission
eas submit --platform ios        # TestFlight
eas submit --platform android    # Play internal testing track
```

## Environment for builds

Set EAS project env vars (Dashboard → project → Environment variables) or
`eas env:push`: all `EXPO_PUBLIC_*` values from `.env.example`
(`EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`,
`EXPO_PUBLIC_API_URL`, `EXPO_PUBLIC_REVENUECAT_*`). Server secrets never go
into mobile builds.

## Device validation checklist (per build)

- [ ] App launches to Welcome with the Vuqiro splash/icon
- [ ] Sign up / sign in against Supabase
- [ ] Feed plays video (native player), scrolling stays smooth
- [ ] Upload flow: pick → upload → processing → ready
- [ ] RevenueCat sandbox: offering prices load from the store; a sandbox
      subscription and a coin pack purchase complete; restore purchases works
- [ ] Permission prompts appear with the configured copy (photos/camera/mic)
- [ ] Locked content: unlock + subscribe paths
- [ ] Settings: account deletion request, restore purchases, legal pages
- [ ] No crash in a 10-minute session (watch Sentry once wired)

## Why builds are not run from this repo

EAS builds execute on Expo's build farm and require the owner's
authenticated Expo/Apple/Google accounts. Everything up to that
authentication boundary is validated here (doctor + prebuild). Running the
commands above is the next owner action; no code changes are expected.
